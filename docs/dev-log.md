# 專案開發日誌 (Development Log & Weekly Report)

這份文件用於紀錄每週的開發進度、技術決策、議題討論以及下週的計畫。旨在提供透明的開發過程，並作為週會報告的基礎。

## [2026-02-26] - 當週開發進度 (Week 1)

### 🚀 當週交付價值 (Deliverables)
- **群組詳情頁 (Group Detail) 重構**：
  - 實作了 **Tab 標籤頁功能**，將資料區分為「人員列表」、「活動列表」及「群組分析」，提升資訊層次感。
  - 整合了 **人員匯入功能**，並針對群組詳情頁優化：**自動預選當前群組**且**鎖定為唯讀**，確保資料匯入的準確性。
- **狀態管理優化**：
  - 將群組與人員對應的「狀態」顯示由標籤改為 **Switch (開關)** 樣式，並對接 `editStatus` API 實現即時同步。
- **目錄結構重構 (Feature-based Structure)**：
  - 將 `member` 模組重構為與 `group` 一致的 `list/detail` 分層結構，提升專案維護性。
- **全動態載入感官 (Loading States)**：
  - 在所有涉及 API 呼叫的頁面（列表、詳情、彈窗）實作 Loading 提示，確保使用者在資料傳輸期間有明確反饋。
- **群組新增功能 (Group Creation)**：
  - 實作了「新增群組」彈窗，支援群組名稱（必填）與備註輸入。
  - 串接 `/edm/group/create` API，並在建立成功後自動重新整理主列表。
- **人員通訊資料編輯完善**：
  - 完成了「行動電話」與「電子郵件」的編輯功能，對接 `editMobile` 與 `editEmail` API，支援非同步更新與錯誤回饋。

### 🐞 技術議題與修復 (Issues & Fixes)
- **API 匯出問題**：修復了 `api/member.ts` 中遺漏 `updateMemberEmailApi` 與 `updateMemberMobileApi` 匯出導致的執行階段錯誤。
- **資料類型匹配**：優化了匯入視窗的 ID 匹配邏輯，處理了字串與數字類型不一致導致無法自動選中群組的問題。

### 💡 技術決策 (Decisions)
- **API 設計**：通訊資料更新採用「細粒度 API」（獨立的 editMobile/editEmail），而非合併式 API，以換取更好的程式碼可讀性與測試便利性。

### 📅 下週提案 (Next Week's Proposal)
1. **管理功能優化**：完善「人員管理」及「群組管理」之核心操作功能。
2. **活動管理系統 (Event Management)**：
   - 實作「建立活動」與「活動列表」功能的完整流程。
   - **互動式關聯**：建立活動與人員、群組間的關聯邏輯（例如：在活動內選定目標群組或特定人員）。
3. **系統整合與合併 (HWS Merge)**：
   - 將本專案與 **HWS 系統** 進行合併，部署至測試機供使用者操作。
   - 確保使用者在未來開發過程中，能同時驗證上週已完成的功能。
4. **安全驗證 (AUTH) 整合**：
   - 將本專案的 **AUTH (登入狀態)** 與 HWS 系統進行深層綁定。
   - 實現多套系統共用帳號密碼機制，並處理相關的資安與權限控管議題。
